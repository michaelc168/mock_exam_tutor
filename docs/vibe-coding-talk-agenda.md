# Vibe Coding 演講大綱

## 演講資訊
- **主題**：Vibe Coding - 用說話的方式讓 AI 幫你做事
- **目標聽眾**：QA、UI 設計師、行銷、產品經理等（非工程師為主）
- **案例項目**：私中入學模擬考試出題系統
- **建議時長**：30-40 分鐘

---

## Agenda

### 1. 開場：一個比喻（3 分鐘）

> 想像你有一個超級助理，他什麼都會做，但需要你「說清楚」你要什麼。
> 
> - 你不需要知道「怎麼做」
> - 你只需要知道「要什麼」
> - 這就是 Vibe Coding

**類比**：
| 傳統方式 | Vibe Coding |
|----------|-------------|
| 自己下廚 | 跟主廚說你想吃什麼 |
| 學 Photoshop | 告訴設計師你的想法 |
| 寫程式 | 用自然語言描述需求 |

---

### 2. 這跟我有什麼關係？（5 分鐘）

#### 各角色的應用場景

| 角色 | 以前 | 現在可以... |
|------|------|-------------|
| **QA** | 請工程師寫測試腳本 | 直接描述測試情境，AI 幫你生成 |
| **UI 設計師** | 只能出設計稿 | 描述互動效果，AI 生成 prototype |
| **行銷** | 請人做數據報表 | 描述要看的指標，AI 幫你整理 |
| **產品經理** | 寫 spec 給工程師 | 直接跟 AI 做出 MVP 驗證想法 |

**關鍵訊息**：不是取代工程師，而是**降低溝通成本**

---

### 3. Live Demo：看我怎麼「聊」出一份考卷（10 分鐘）

#### 場景：幫小孩準備私中入學考試

**第一句話**：
> 「幫我出一份模擬考，國語 20 題、英文 20 題、數學 40 題」

**AI 做的事**（觀眾不需要懂技術）：
1. 📖 讀取題庫（500+ 題）
2. 🎲 生成「類似但不同」的新題目
3. 📝 整理成完整試卷
4. ✅ 附上答案和解析

**結果**：1 分鐘內生成 80 題試卷

#### 延伸對話：
> 「轉成 PDF，數學公式要漂亮」

**這就是 Vibe** — 我沒說要用什麼技術，只說「要漂亮」

---

### 4. 真實故事：AI 也會出錯（7 分鐘）

#### 問題一：數學公式顯示亂碼

我說：「數學公式要漂亮」

AI 做了，但結果：`$\frac{5}{8}$` ← 這是什麼鬼？

我的反應：
> 「你在開玩笑？你看 76 題、78 題。」

#### 問題二：選項 (C) 變成 ©

AI 又「修好了」，結果：

- (A) 正常
- (B) 正常
- © ← 變成版權符號了！

我的反應：
> 「(C) 怎麼會變成 copyright 的 ©？？？」

#### 重點

| 學到的事 | 說明 |
|----------|------|
| AI 不是魔法 | 會犯錯，而且錯得很離譜 |
| 要驗收 | 不能說「我改好了」就相信 |
| 要明確 | 說清楚「哪裡錯」比「幫我修」有用 |
| 要耐心 | 來回 4 次才真正修好 |

### 5. 什麼時候該用？什麼時候不該用？（5 分鐘）

#### ✅ 適合的情境

| 情境 | 例子 |
|------|------|
| **探索想法** | 「幫我想 5 個活動標題」 |
| **快速原型** | 「做一個簡單的報名表單」 |
| **重複性工作** | 「把這 100 筆資料整理成表格」 |
| **學習新工具** | 「教我怎麼用 Figma 做動畫」 |

#### ❌ 不適合的情境

| 情境 | 原因 |
|------|------|
| **機密資料** | AI 可能記住你的內容 |
| **需要 100% 正確** | AI 會「自信地」說錯 |
| **複雜商業邏輯** | 需要人類判斷和經驗 |
| **法律/財務文件** | 出錯代價太高 |

---

### 6. 三個馬上能用的技巧（5 分鐘）

#### 技巧 1：說清楚「是什麼」而不是「不是什麼」

- ❌ 「不要太複雜」
- ✅ 「用小學生能懂的詞彙」

#### 技巧 2：給例子

- ❌ 「幫我寫一封感謝信」
- ✅ 「幫我寫一封感謝信，風格像這樣：[貼上範例]」

#### 技巧 3：分步驟要求

- ❌ 「幫我做完整的行銷企劃」
- ✅ 「先給我 3 個主題方向」→「第 2 個展開細節」→「加上預算估算」

---

### 7. Q&A（5 分鐘）

**預設問題**：

1. **這樣工程師不就失業了？**
   → 不會，反而需要更多人來「驗收」和「優化」AI 的輸出

2. **我完全不懂程式，也能用嗎？**
   → 可以，但要學會「怎麼問問題」

3. **有免費的工具嗎？**
   → ChatGPT 免費版、Claude、Cursor（工程向）

4. **怎麼知道 AI 的回答對不對？**
   → 這正是你的價值：用專業判斷驗收

---

## 總結金句

> **Vibe Coding 不是讓你變成工程師，**
> **而是讓你能「跟工程師等級的 AI」合作。**

你帶來的是：
- 🎯 **判斷力** — 知道要做什麼
- 🔍 **驗收能力** — 知道做得對不對
- 💬 **溝通能力** — 知道怎麼說清楚

AI 帶來的是：
- ⚡ **執行速度**
- 📚 **知識廣度**
- 🔄 **不會累**
